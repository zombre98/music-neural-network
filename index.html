Code

For this project, we use GitHub, here is the link to our public repository:

  • https://github.com/zombre98/music-neural-network

Like our organization the code is divided into 3 parts: Features, LSTM and CNN.
In the app folder you will find the 3 parts.
The features folder which implements a Python MidiData class. This class allows us to parse the files of our dataset and extract features that our artificial intelligences can learn. To do this there is a TypeData subclass that offers an interface to extract notes or tempos.
The LSTM folder contains 2 notebooks that allow us to test our LSTM model faster! The file LSTM_over_notes.ipynb and the file LSTM_on_binary.ipynb these two implementation of 2 model LSTM try for one to recognize the music thanks to the notes of the file extracted from the MidiData classes. The second file is an attempt to implement an LSTM which analyzes the file just by reading it without any analysis. Regarding the .html and .py files that are in this folder they are just files generated by our notebook to be able to read us and share our code
The CNN folder is organized in the same way as the LSTM folder it also contains an exploration of another model of Artificial Intelligence. We also find there the file CNN_over_notes.ipynb which is a notebook to be able to easily launch new Artificial Inteligence configuration it contains a model that tries to analyze the notes of a music! Regarding the organization of the rest of the folder we decided to push the dataset with the files that allow us to read it with maestro-v2.0.0.json.

Result

The goal of this project was to explore the classification of music through several models of Artificial Intelligence. Some of the results are not very convincing since their use is not useful in this kind of problem. The results are very different between Artificial Intelligence since their implementation is very different depending on their structure, their strengths and weaknesses.
