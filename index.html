Code

For this project, we use GitHub, here is the link to our public repository:

  • https://github.com/zombre98/music-neural-network

Like our organization the code is divided into 3 parts: Features, LSTM and CNN.

File name Description
app/ This folder you will find the 3 parts: Features, CNN and LSTM
app/features/ This folder contains the class that parse midi files
app/features/MidiData.py This class allows us to parse the files of our dataset and extract features that our artificial intelligences can learn. To do this there is a TypeData.py subclass that offers an interface to extract notes or tempos.
app/lstm It contains 2 jupyter notebooks that allow us to test our LSTM model faster
app/lstm/LSTM_over_notes.ipynb This file implements an AI which tries to recognize the music thanks to the notes extracted from the file with the MidiData class.
app/lstm/LSTM_on_binary.ipynb This file implements an AI that tries to recognize the music by reading the binary files without extracting any features.
app/cnn/ This folder is organized in the same way as the LSTM folder it also contains an exploration of another model of Artificial Intelligence
app/cnn/CNN_over_notes.ipynb This file implements a convolutional neural network that tries to learn the music score by browsing the notes.
data/ This folder is our dataset which contains the file maestro-v2.0.0.json that allow us to read it

All the .html and .py files that are in this folder they are just files generated by our notebook to be able to read us and share our code

Result

The goal of this project was to explore the classification of music through several models of Artificial Intelligence. Some of the results are not very convincing since their use is not useful in this kind of problem. The results are very different between Artificial Intelligence since their implementation is very different depending on their structure, their strengths and weaknesses.
